Innehåll på sajten byggs av WordPress-sidor med Gutenberg-blocks. Pluginet och temat lägger till egna block och mönster som kan användas tillsammans med standardblocken från WordPress.

Site content is built from WordPress pages using Gutenberg blocks. The plugin and theme add their own blocks and patterns that work alongside WordPress's standard blocks.

Block vs mönster

Blocks vs patterns

  • Ett block är en enskild byggsten — t.ex. en rubrik, en bild, eller bildkarusellen. Block hittas under Block-fliken i infogaren.
  • A block is a single building unit — e.g. a heading, an image, or the image carousel. Blocks live under the Blocks tab in the inserter.
  • Ett mönster är en färdig sammansättning av flera block. När det infogas blir det till de underliggande blocken, fritt redigerbara. Mönster hittas under Mönster-fliken.
  • A pattern is a pre-arranged set of blocks. Once inserted, it becomes those individual blocks, freely editable. Patterns live under the Patterns tab.

Stub-sidor som temat skapar

Stub pages the theme creates

Vid aktivering skapar temat ett antal start-sidor (t.ex. Hem, Nyheter, Kalender, Träning). Sidornas innehåll är fritt redigerbart — temat skriver bara över innehållet vid första aktiveringen, aldrig efteråt.

On activation, the theme creates a set of stub pages (e.g. Hem, Nyheter, Kalender, Träning). Their content is freely editable — the theme writes the initial content on first activation only, never again.

Hero-mönster

Hero pattern

Hero-bannern på framsidan är ett mönster som heter Landing Hero. Det består av en omslagsblock (cover) med rubrik, underrubrik och två knappar. Det går att infoga på vilken sida som helst — i infogaren, gå till fliken Mönster, kategorin Rockaden, och välj Landing Hero. Bilden sätts via cover-blockets verktygsfält efter infogning, och text/knappar redigeras direkt i blocken.

The front-page hero banner is a pattern called Landing Hero. It's a cover block with a heading, subtitle, and two buttons. It can be inserted on any page — in the inserter, go to the Patterns tab, the Rockaden category, and pick Landing Hero. The image is set via the cover block's toolbar after insertion, and the text/buttons are edited directly inside the blocks.

Tips: Standardtexter och knapplänkar i hero-mönstret är skrivna för framsidan ("Bli medlem", "Prova gratis"). Tänk på att skriva om dem för andra sidor. Tip: The hero pattern's default text and button targets are written for the front page ("Bli medlem", "Prova gratis"). Remember to rewrite them on other pages.

Sidor med sidopanel (Page Section Nav-mallen)

Pages with a sidebar (Page Section Nav template)

Sidor som /medlemmar/ har en automatisk sidopanel till vänster med navigering till undersidor. Detta drivs av två saker som måste sättas på rätt sätt: sidhierarkin (förälder/barn-relationer mellan WP-sidor) och sidmallen Page (Section Nav). Sidopanelen byggs sedan automatiskt av blocket Section Nav.

Pages like /medlemmar/ have an automatic left sidebar that navigates between sub-pages. This is driven by two things that need to be set correctly: the page hierarchy (parent/child relationships between WP pages) and the Page (Section Nav) template. The sidebar itself is then built automatically by the Section Nav block.

Så här bygger du en sektion med sidopanel

How to build a sidebar section

  1. Skapa rotssidan (t.ex. "Medlemmar") som en vanlig sida.
  2. Create the root page (e.g. "Members") as a regular page.
  3. Skapa undersidor och sätt Sidan → Sidattribut → Förälder till rotssidan. Detta är det enda sättet undersidor "syns" som tillhörande sektionen — själva URL-sökvägen räcker inte.
  4. Create sub-pages and set Page → Page Attributes → Parent to the root page. This is the only way sub-pages are seen as belonging to the section — having a similar URL path is not enough.
  5. Tilldela mallen Page (Section Nav) på varje sida i sektionen (både rot och undersidor): Sidan → Mall. Glömmer du mallen renderas sidan utan sidopanel.
  6. Assign the template Page (Section Nav) on each page in the section (root and sub-pages): Page → Template. If you forget the template, the page renders without the sidebar.
  7. Publicera/uppdatera. Sidopanelen byggs automatiskt nästa gång sidan laddas.
  8. Publish/update. The sidebar is built automatically the next time the page loads.

Vad sidopanelen visar

What the sidebar shows

Section Nav-blocket utgår från den aktuella sidan och hittar roten i sektionen — antingen sidans förälder, eller sidan själv om den inte har någon förälder. Sidopanelen listar sedan roten överst följt av alla direkta undersidor, sorterade efter Sidattribut → Ordning och sedan titel. Den nuvarande sidan markeras automatiskt.

The Section Nav block starts from the current page and finds the root of the section — either the page's parent, or the page itself if it has no parent. The sidebar then lists the root at the top followed by all direct sub-pages, sorted by Page Attributes → Order and then title. The current page is highlighted automatically.

Detta innebär att samma rotssida och samma syskonlista visas oavsett vilken sida i sektionen besökaren är på.

This means the same root and the same sibling list are shown regardless of which page within the section the visitor is on.

Att tänka på

Things to know

  • Bara en nivå. Sidopanelen visar roten och dess direkta undersidor. Barnbarn (undersidor till undersidor) visas inte.
  • One level only. The sidebar shows the root and its direct sub-pages. Grandchildren (sub-pages of sub-pages) are not listed.
  • Inga barn = ingen panel. Om rotssidan inte har några undersidor visas ingen sidopanel alls. Lägg till minst en undersida (eller välj en annan sidmall).
  • No children = no sidebar. If the root page has no sub-pages, no sidebar is shown at all. Add at least one sub-page (or use a different template).
  • Ordning kontrolleras via Sidattribut. För att styra i vilken ordning länkarna visas, sätt ett heltal i Sidattribut → Ordning på varje undersida (lägre tal först).
  • Order is controlled via Page Attributes. To control the order of the links, set an integer in Page Attributes → Order on each sub-page (lower numbers first).
  • URL-strukturen. Med förälder satt blir undersidans URL /rotssida/undersida/. Detta hanteras automatiskt av WordPress.
  • URL structure. With a parent set, the sub-page's URL becomes /root-page/sub-page/. WordPress handles this automatically.
Tips: När du publicerar en ny undersida och sidopanelen inte uppdaterar sig direkt — verifiera att (a) Förälder är satt, (b) Page (Section Nav)-mallen är vald på både rot och undersida, och (c) statusen är Publicerad (inte Utkast). Tip: When you publish a new sub-page and the sidebar doesn't update — verify that (a) Parent is set, (b) Page (Section Nav) template is selected on both the root and the sub-page, and (c) the status is Published (not Draft).

Övriga mönster

Other patterns

Mönstren Landing Hero, Landing Why Section och Landing News & Shop utgör tillsammans framsidans layout och används av temat för att bygga /hem vid aktivering. De kan infogas på andra sidor om man vill ha samma utseende, men är primärt avsedda för framsidan.

The Landing Hero, Landing Why Section, and Landing News & Shop patterns together form the front-page layout; the theme uses them to build /hem on activation. They can be inserted on other pages if you want the same look, but they're primarily intended for the front page.
